## Version Control Workflow

The project was managed using Git and GitHub following best industry practices.

### Branching Strategy
- main (stable code)
- feature/ui-polish
- feature/mini-game-fixes
- feature/auth-improvements
- bugfix/typo-and-small-bugs

Each branch focused on a specific task.

### Pull Requests & Merging
For each branch:
1. Code changes were committed.
2. A Pull Request (PR) was opened.
3. PR was reviewed and merged into main.
4. Branch was deleted after merging.

### Release Tag
The final project build was published under tag **v1.0.0** in GitHub Releases.
